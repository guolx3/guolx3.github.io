事务管理

发表于 2019-09-27 | 分类于 数据库 |

概念

事务是构成单一逻辑工作的一系列操作的集合

ACID

事务具有以下特性：

  • 原子性Atomicity：事务不可分割，要么全部执行，要么全部不执行
  • 一致性Consistency：事务必须保证数据库的一致性
  • 隔离性Isolation：多个事务并发执行时，系统保证每个事务都感觉不到其他事务正在并发执行
  • 持久性Durability：事务成功完成后对数据库的改变会保持下去

抽象事务模型

事务处于以下状态之一：

  • 活动的active
  • 部分提交的partially committed
  • 失败的failed
  • 终止的aborted
  • 提交的committed

回滚：

  1. 杀死：逻辑错误
  2. 重启：硬件错误

可串行化

如果强制事务串行地执行，那么很容易就能保持一致性，而并发很可能违背隔离性。
如果在并发执行中，我们能保证所执行的任何调度的效果都跟没有并发执行的调度效果一样，就可以确保数据库的一致性。
这种调度称为可串行化调度

术语

  • 调度：指令在系统中执行的时间顺序

  • 串行调度：同一事务的各个指令紧挨在一起（执行完一个事务再执行下一个事务),如下图的调度$S$

  • 冲突：如果指令$I$和$J$是在相同的数据上操作，并且其中至少有一个是写操作，则称$I$和$J$是冲突的

  • 冲突等价：如果调度$S$经过一些列非冲突指令交换转换成$S’$，则称$S$和$S’$冲突等价

  • 冲突可串行化：如果调度$S$与一个串行调度冲突等价，则称$S$是冲突可串行化的

如何判断调度$S$是否冲突可串行化

构造优先图$G = (V, E)$, $V$是顶点集，$E$是边集
边集由满足下列三个条件之一的$T_i \rightarrow T_j$组成：

  • $T_i$执行$write(Q)$后，$T_j$执行$read(Q)$
  • $T_i$执行$read(Q)$后，$T_j$执行$write(Q)$
  • $T_i$执行$write(Q)$后，$T_j$执行$write(Q)$

可恢复和无级联调度

可恢复调度

当并发执行中事务发生故障，有些调度不能恢复。

例如下图的调度

如果T6在T7提交后发生故障，需要回滚；
而T7读取了T6写入的数据（我们可以说T7依赖于T6），也需要中止；
但T7已经提交了，无法中止。
这样的调度是不可恢复调度

可恢复调度应该满足：对于每对事务T1和T2，如果T2读取了之前由T1写入的数据，那么T1应该先于T2提交。

无级联调度

即使一个调度是可恢复的，如果多对事务之间存在依赖关系，可能引发级联回滚，导致大量的撤销工作。

无级联调度应该满足:对于每对事务T1和T2，如果T2读取了之前由T1写入的数据，那么T1应该在T2这一读操作之前提交

事务隔离性级别

隔离性级别由高到低由：

  1. 可串行化：保证可串行化调度。
  2. 可重复读：只能读取已提交的数据，并且在一个事务两次读取一个数据时，其他事务不得更新该数据。不要求与其他事务可串行化，
  3. 已提交读：只能读取已提交的数据。不要求可重复读。
  4. 未提交读：允许读取未提交的数据。

我们可以通过多种并发控制机制来实现事务的隔离性级别

C++中的三种参数传递方式

发表于 2019-09-27 |

值传递

值传递会将实参拷贝一份副本作为局部变量传进函数，在函数内修改该副本不会改变原来实参的值

指针传递

指针传递本质上也是值传递，只是传递的值是指针变量存储的地址值。修改该地址值指向的变量可以修改变量的值

引用传递

形参相当于实参的别名，对形参的操作就是对实参的操作。

指针的引用的区别

  1. 指针本质是一个变量，在逻辑上时独立的，引用时某个变量的别名，在逻辑上依附与某个变量，在定义引用时必须初始化

  2. sizeof(引用)得到的时引用变量的大小，sizeof(指针)得到的时指针变量的大小

  3. 编译时指针在符号表对应的地址值时指针变量的地址值，引用对用的是引用对象的地址值

树和二叉树

发表于 2019-09-27 |

定义

树是$n$个结点的有限集。在任意一颗非空树中：

  1. 有且仅有一个根节点
  2. 其余节点可分为$m$个互不相交的有限集
  3. 其中每个集合本身也是一颗树，称为跟的子树（递归定义）。

基本术语

度

结点拥有的子树数称为结点的度

叶子结点

度为$0$的结点称为叶子结点

内部结点

除了根结点和叶子结点外的其他结点称为内部结点

孩子/双亲/兄弟/祖先/子孙

  1. 结点的子树的根称为该结点的孩子，该结点被称为孩子的双亲
  2. 同一结点的孩子之间称为兄弟
  3. 从跟到该结点所经分支上的所有结点称为该结点的祖先。

层次

根为第一层，根的孩子为第二层。
若某结点在第$i$层，则其孩子在第$i + 1$层

深度/高度

树中结点的最大层次称为树的深度或高度

C++的内联函数

发表于 2019-09-22 | 分类于 学习笔记 |

在函数体前加上inline关键字可以将函数设置为内联函数。
在编译时，编译器会将调用内联函数的地方替换成内联函数体。
内联函数的作用是用空间换时间，牺牲一定的内存来节省函数调用的开销。

注意

  1. 内联函数中不能使用循环和switch
  2. 内联函数不能递归调用自己
  3. 内联函数不能调用其他函数
  4. 代码行数较少且调用频繁的函数推荐使用内联函数
  5. inline关键字只是对编译器的一种建议，被inline修饰的函数不一定被内联。如果函数体过大，编译器会放弃内联而把代码当成普通函数

与宏定义的区别

  • 宏在编译之前完成文本替换，内联函数在编译时被处理
  • 内联函数本质是函数，可以像普通函数一样调用，不像宏定义那样需要加上括号来防止歧义
  • 最好用内联函数来代替宏定义

内存管理4——虚拟内存

发表于 2019-09-17 | 分类于 学习笔记 |

之前讨论的几种管理方式都具有一下两种特征：

  • 一次性。 作业必须一次性全部装入内存后才开始运行，导致当作业过大时不能被全部装入
  • 驻留性。 作业装入内存后，一直驻留在内存中，运行中的进程会因为等待I/O而被阻塞，可能长期处于等待状态

为此，可以采用虚存技术，使得在进程运行时并不需要所有页（段）都在内存中，只在需要的时候读取页（段）。

虚拟内存的实现需要一定的硬件支持，一般包括：

  • 一定容量的内存和外存。
  • 页表机制（段表机制）。
  • 中断机构。当用户程序要访问的部分尚未调入内存时，则产生中断
  • 地址变换机构。逻辑地址到物理地址的转换

局部性原理

  • 时间局部性。 执行的指令可能在不久后再次执行，被访问的数据可能在不久后再次被访问。 原因之一时程序中存在大量的循环语句
  • 空间局部性。 一旦程序访问了某存储单元，则其附近的存储单元也可能被访问。 原因之一时指令通常时顺序存放的，数据也一般是以数组，表等形式存在

请求分页管理方式

请求分页管理方式在基本分页管理方式的基础上增加了请求调页功能和页面置换功能。
这是目前最常用的一种实现虚拟存储器的方法

页表机制

  • 状态位P：指示该页是否已经调入内存
  • 访问字段A：记录访问相关的信息，例如一段时间内被访问的次数，共置换算法参考
  • 修改位M：标识该页在调入内存后是否被修改过
  • 外存地址：用于指出该页在外存的地址，通常时物理块号。

缺页中断机制

每当要访问的页面不在内存中时，就产生一次缺页中断。

内存管理2——连续分配管理方式

发表于 2019-09-15 | 分类于 学习笔记 |

单一连续分配

将内存分为系统区和用户区。系统区供操作系统使用，处于低地址。用户区是除了系统区外的内存空间。
优点

  • 简单
  • 无外部碎片

缺点

  • 只适合单用户的操作系统
  • 有内部碎片
  • 存储器的利用率低

固定分区

最简单的多道程序存储管理方式。
将内存划分成若干个固定大小的区域，区域大小可以相等或者按规则递增，每个区域只能放入一个程序。
为了方便内存分配，需要一张分区说明表，每行记录分区的大小、起始地址和分配状态。分配内存时查找该表寻找合适的分区，如果没找到则不分配内存

优点

  • 简单
  • 处理开销小

缺点

  • 有内部碎片
  • 不能实现多进程共享一个主存区

目前基本没有操作系统使用固定分区的方法

动态分区

不预先划分内存，而是在进程装入内存时根据进程的大小动态建立分区，使得分区大小刚好适合进程大小

缺点

  • 时间长了有外部碎片，可以用紧凑技术对进程进行移动和整理

分配策略

如果同时又多个空闲块满足要求，需要采用一定的分配策略来决定分配哪一块

  1. 首次适应First Fit。按照地址顺序查找到的第一个合适分区。最简单、最好、最快。但是低地址会出现小的空闲分区，而每次重头开始查找都会经过这些分区，增加了查找的开销。
  2. 最佳适应Best Fit。将空闲分区按照容量递增排序，顺序查找到的第一个合适分区。
  3. 最坏适应Worst Fit。将空闲分区按照容量递增排序，顺序查找到的第一个合适分区。
  4. 邻近适应Next Fit。由首次适应演变而来，不同之处是从上一次查找结束的位置开始继续查找。它试图解决首次适应算法低地址空闲分区的问题，但会导致末尾的最大空闲块被分裂成小碎片。

好的程度：首次适应>邻近适应>最佳适应>最坏适应

以上三种分区管理方式的共同点是用户进程在主存中都是连续存储的

内存管理1——基本概念

发表于 2019-09-15 | 分类于 学习笔记 |

为什么要进行内存管理

不可能将所有的进程和所需的资源全都放入内存，所以需要操作系统对内存进行合理的划分和动态管理。

内存管理的功能

  1. 内存空间的分配和回收
  2. 地址转换，将逻辑地址转换为物理地址
  3. 内存的扩展，利用虚拟存储技术或自动覆盖技术从逻辑上扩展内存
  4. 保护，保证各道作业互不干扰

内存管理的基本原理和要求

程序的装入和链接

将源程序转换为可以在内存中执行的进程有以下3步：

  1. 编译。将源程序编译成若干目标模块
  2. 链接。将若干模块以及需要的库函数链接成一个完整的装入模块
  3. 装入。将装入模块装入内存

链接

按照链接的时机不同，可以分成：

  • 静态链接
  • 装入时链接
  • 运行时链接

装入

  • 绝对装入。已经知道程序应该装入的绝对地址，按照地址装入程序和数据，只适用于单道程序设计。该地址可以在编译或汇编时指出或由程序员赋予
  • 可重定位装入。多道程序环境中进程的起始地址通常为0，其他地址是相对地址；在装入时根据相对地址进行地址变换，一次分配完整的内存，如果内存不够则不装入。运行中不能移动或申请内存。
  • 动态装入，将地址转换推迟到运行时再进行。需要重定位寄存器的支持，将内存分配到不连续的存储区，运行时可以动态申请分配内存，可以提供一个比存储空间大的地址空间。

逻辑地址和物理地址

每个目标模块从0开始编址，这些相对地址即逻辑地址；
物理地址是内存中物理单元的真实地址；
将逻辑地址转换为物理地址称为地址重定位

内存保护

  • 保护操作系统不受用户进程的影响
  • 保护用户进程不受其他用户进程的影响

内存保护有两种方法：

  • 采用上下限寄存器记录用户作业在主存中的上下限地址，每次访问地址都进行比较判断是否越界
  • 采用重定位寄存器（基址寄存器）和界地址寄存器（限长寄存器）。重定位寄存器记录最小物理地址，界地址寄存器记录最大物理地址，每次都于界地址寄存器进行比较，如果没有越界则加上重定位寄存器的值映射成物理地址

扩充内存的方法——覆盖和交换

覆盖

程序运行时不可能每时每刻都用到程序和数据的所有部分，所以我们可以把经常访问的部分放在固定区，其它的按照调用关系分段，要用的时候再调入覆盖区

交换

交换有两个基本操作：

  • 换出：将处于等待状态的程序从内存移到辅存
  • 换入：将就绪的程序从辅存移到内存

覆盖主要用于同一个程序，交换用于不同的进程
