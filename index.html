C++中的类型转换

在某些情况下，类型转换会自动执行无需程序员借入，这种类型转换称为隐式转换；程序员也可以显式的将对象转换为另一种类型，这种方法称为强制类型转换cast

隐式转换

发生隐式转换的场景有：

  1. 初始化过程中，初始值转换成变量的类型；赋值语句中，右侧运算对象转换称左侧运算对象类型。 

    bool b = 42; // b为真
int i = b; // i = 1
i = 3.14; // i = 3
double f = i; // f = 3.0
unsigned int c = -1; // c = 4294967295
int d = 4294967296; // warning: overflow in implicit constant conversion

    • 当赋给无符号类型的初始值超出范围时，它的实际值是初始值对该类型总大小取模之后的余数。若$a < 0$, $a mod b = a % b + b$
    • 当赋给有符号类型的初始值超出范围时，它的实际值是未定义的

  2. 整数提升。将小整数类型转换成较大的整数类型。转换后的类型要能容纳原类型所有可能的值

  3. 在条件中，非布尔值转换成布尔类型
  4. 如果算术运算或关系运算有多种类型则转换成同一种类型，尽量不损失精度
  5. 数组转成指针。 在大多数数组的表达式中，数组自动转换成指向数组首元素的指针 
    int ia[10];
int* ip = ia; //ia为指向数组首个元素的指针
    注意，当数组被用作decltype的参数，或者作为取地址符&，sizeof及typeid的运算对象时，上述转换不会发生。 
    int ia[10];
cout << typeid(ia).name() << endl; // A10_i
  6. 转换成常量。允许指向非常量的指针和引用，允许转换成指向相应的常量类型的指针。 
    int i = 3;
const int& j = i;
j = 5; // error: assignment of read-only reference 'j'
i = 5; // 不报错

基本数据类型

C++的基本数据类型包括算术类型和空类型

算术类型

算数类型所占的比特数在不同机器上会有所差别。 C++标准规定了尺寸的最小值，允许编译器赋予这些类型更大的尺寸

C++ 语言规定：

  • int至少和short一样大
  • long至少和int一样大
  • long long时C++11新定义的，至少和long一样大

带符号和无符号类型

除了bool，其他整型可以分成带符号的signed和无符号unsigned两种。

  • signed: 正数，零，负数
  • unsigned: 仅表示大于或等于零的值

int,short,long和long long默认是带符号的，通过在类型名前添加unsigned可以得到无符号类型。

选择类型的一些建议

  1. 使用int进行整数运算。 short常常太小，long常常跟int一样大。如果超过了int的范围，选用long long
  2. 算数表达式中不要使用char和bool
  3. 使用double进行浮点运算。 float通常精度不够，且单精度和双精度的计算代价相差不大

基于锁的协议

确保隔离性的方法之一就是对数据项的访问以互斥的方式进行。
实现这种需求的方法是只允许事务访问该事务持有锁的数据

锁

  • 共享锁$S$，如果事务$T_i$获得数据$Q$的共享锁，则$T_i$可读但不可写$Q$。其他事务可以给$Q$加共享锁，不能加排他锁
  • 排他锁$X$，如果事务$T_i$获得数据$Q$的排他锁，则$T_i$可读且可写$Q$。其他事务不能给$Q$加锁

锁的授予

  1. 事务向并发控制管理器申请适当的锁
  2. 如果该数据项已经被加上了不相容的锁，则该事务等待，直到所有其他事务持有的不相容锁被释放
  3. 只有并发管理控制器授予事务所需锁后事务才能继续其操作

两阶段封锁协议

保证可串行性的一个协议，要求每个事务分两个阶段提出加锁和解锁申请。

  1. 增长阶段：事务可以获得锁，但不能释放锁
  2. 缩减阶段：事务可以释放锁，但不能获得新锁

在调度中该事务最后加锁的位置（增长阶段的结束点）称为事务的封锁点，多个事务可以根据封锁点进行排序，这个顺序就是事务的一个可串行化顺序。

两阶段封锁协议不能保证不发生死锁。

级联回滚的问题可以通过严格两阶段封锁协议避免。 处理两阶段封锁协议的要求，它还要求事务持有的所有排他锁必须在事务提交后才能释放。
这样可以防止其他事务在未提交前读这些数据。

事务管理

发表于 2019-09-27 | 分类于 数据库 |

概念

事务是构成单一逻辑工作的一系列操作的集合

ACID

事务具有以下特性：

  • 原子性Atomicity：事务不可分割，要么全部执行，要么全部不执行
  • 一致性Consistency：事务必须保证数据库的一致性
  • 隔离性Isolation：多个事务并发执行时，系统保证每个事务都感觉不到其他事务正在并发执行
  • 持久性Durability：事务成功完成后对数据库的改变会保持下去

抽象事务模型

事务处于以下状态之一：

  • 活动的active
  • 部分提交的partially committed
  • 失败的failed
  • 终止的aborted
  • 提交的committed

回滚：

  1. 杀死：逻辑错误
  2. 重启：硬件错误

可串行化

如果强制事务串行地执行，那么很容易就能保持一致性，而并发很可能违背隔离性。
如果在并发执行中，我们能保证所执行的任何调度的效果都跟没有并发执行的调度效果一样，就可以确保数据库的一致性。
这种调度称为可串行化调度

术语

  • 调度：指令在系统中执行的时间顺序

  • 串行调度：同一事务的各个指令紧挨在一起（执行完一个事务再执行下一个事务),如下图的调度$S$

  • 冲突：如果指令$I$和$J$是在相同的数据上操作，并且其中至少有一个是写操作，则称$I$和$J$是冲突的

  • 冲突等价：如果调度$S$经过一些列非冲突指令交换转换成$S’$，则称$S$和$S’$冲突等价

  • 冲突可串行化：如果调度$S$与一个串行调度冲突等价，则称$S$是冲突可串行化的

如何判断调度$S$是否冲突可串行化

构造优先图$G = (V, E)$, $V$是顶点集，$E$是边集
边集由满足下列三个条件之一的$T_i \rightarrow T_j$组成：

  • $T_i$执行$write(Q)$后，$T_j$执行$read(Q)$
  • $T_i$执行$read(Q)$后，$T_j$执行$write(Q)$
  • $T_i$执行$write(Q)$后，$T_j$执行$write(Q)$

可恢复和无级联调度

可恢复调度

当并发执行中事务发生故障，有些调度不能恢复。

例如下图的调度

如果T6在T7提交后发生故障，需要回滚；
而T7读取了T6写入的数据（我们可以说T7依赖于T6），也需要中止；
但T7已经提交了，无法中止。
这样的调度是不可恢复调度

可恢复调度应该满足：对于每对事务T1和T2，如果T2读取了之前由T1写入的数据，那么T1应该先于T2提交。

无级联调度

即使一个调度是可恢复的，如果多对事务之间存在依赖关系，可能引发级联回滚，导致大量的撤销工作。

无级联调度应该满足:对于每对事务T1和T2，如果T2读取了之前由T1写入的数据，那么T1应该在T2这一读操作之前提交

事务隔离性级别

隔离性级别由高到低由：

  1. 可串行化：保证可串行化调度。
  2. 可重复读：只能读取已提交的数据，并且在一个事务两次读取一个数据时，其他事务不得更新该数据。不要求与其他事务可串行化，
  3. 已提交读：只能读取已提交的数据。不要求可重复读。
  4. 未提交读：允许读取未提交的数据。

我们可以通过多种并发控制机制来实现事务的隔离性级别

C++中的三种参数传递方式

发表于 2019-09-27 |

值传递

值传递会将实参拷贝一份副本作为局部变量传进函数，在函数内修改该副本不会改变原来实参的值

指针传递

指针传递本质上也是值传递，只是传递的值是指针变量存储的地址值。修改该地址值指向的变量可以修改变量的值

引用传递

形参相当于实参的别名，对形参的操作就是对实参的操作。

指针的引用的区别

  1. 指针本质是一个变量，在逻辑上时独立的，引用时某个变量的别名，在逻辑上依附与某个变量，在定义引用时必须初始化

  2. sizeof(引用)得到的时引用变量的大小，sizeof(指针)得到的时指针变量的大小

  3. 编译时指针在符号表对应的地址值时指针变量的地址值，引用对用的是引用对象的地址值

树和二叉树

发表于 2019-09-27 |

定义

树是$n$个结点的有限集。在任意一颗非空树中：

  1. 有且仅有一个根节点
  2. 其余节点可分为$m$个互不相交的有限集
  3. 其中每个集合本身也是一颗树，称为跟的子树（递归定义）。

基本术语

度

结点拥有的子树数称为结点的度

叶子结点

度为$0$的结点称为叶子结点

内部结点

除了根结点和叶子结点外的其他结点称为内部结点

孩子/双亲/兄弟/祖先/子孙

  1. 结点的子树的根称为该结点的孩子，该结点被称为孩子的双亲
  2. 同一结点的孩子之间称为兄弟
  3. 从跟到该结点所经分支上的所有结点称为该结点的祖先。

层次

根为第一层，根的孩子为第二层。
若某结点在第$i$层，则其孩子在第$i + 1$层

深度/高度

树中结点的最大层次称为树的深度或高度

C++的内联函数

发表于 2019-09-22 | 分类于 学习笔记 |

在函数体前加上inline关键字可以将函数设置为内联函数。
在编译时，编译器会将调用内联函数的地方替换成内联函数体。
内联函数的作用是用空间换时间，牺牲一定的内存来节省函数调用的开销。

注意

  1. 内联函数中不能使用循环和switch
  2. 内联函数不能递归调用自己
  3. 内联函数不能调用其他函数
  4. 代码行数较少且调用频繁的函数推荐使用内联函数
  5. inline关键字只是对编译器的一种建议，被inline修饰的函数不一定被内联。如果函数体过大，编译器会放弃内联而把代码当成普通函数

与宏定义的区别

  • 宏在编译之前完成文本替换，内联函数在编译时被处理
  • 内联函数本质是函数，可以像普通函数一样调用，不像宏定义那样需要加上括号来防止歧义
  • 最好用内联函数来代替宏定义

内存管理4——虚拟内存

发表于 2019-09-17 | 分类于 学习笔记 |

之前讨论的几种管理方式都具有一下两种特征：

  • 一次性。 作业必须一次性全部装入内存后才开始运行，导致当作业过大时不能被全部装入
  • 驻留性。 作业装入内存后，一直驻留在内存中，运行中的进程会因为等待I/O而被阻塞，可能长期处于等待状态

为此，可以采用虚存技术，使得在进程运行时并不需要所有页（段）都在内存中，只在需要的时候读取页（段）。

虚拟内存的实现需要一定的硬件支持，一般包括：

  • 一定容量的内存和外存。
  • 页表机制（段表机制）。
  • 中断机构。当用户程序要访问的部分尚未调入内存时，则产生中断
  • 地址变换机构。逻辑地址到物理地址的转换

局部性原理

  • 时间局部性。 执行的指令可能在不久后再次执行，被访问的数据可能在不久后再次被访问。 原因之一时程序中存在大量的循环语句
  • 空间局部性。 一旦程序访问了某存储单元，则其附近的存储单元也可能被访问。 原因之一时指令通常时顺序存放的，数据也一般是以数组，表等形式存在

请求分页管理方式

请求分页管理方式在基本分页管理方式的基础上增加了请求调页功能和页面置换功能。
这是目前最常用的一种实现虚拟存储器的方法

页表机制

  • 状态位P：指示该页是否已经调入内存
  • 访问字段A：记录访问相关的信息，例如一段时间内被访问的次数，共置换算法参考
  • 修改位M：标识该页在调入内存后是否被修改过
  • 外存地址：用于指出该页在外存的地址，通常时物理块号。

缺页中断机制

每当要访问的页面不在内存中时，就产生一次缺页中断。
