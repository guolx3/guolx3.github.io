C++中的运算符

算术运算符

  • %取余运算符的运算对象必须是整数类型
  • C++11规定商一律向0取整
  • 算术运算符都满足左结合律，即当优先级相同时从左到右计算

逻辑和关系运算符

短路求值：

  • && 只有当左侧为真才计算右侧
  • || 只有当左侧为假才计算右侧

赋值运算符

  • 赋值运算符左侧运算对象必须是一个可以修改的左值
  • 赋值运算符满足右结合律

递增和递减运算符

即++和--，有前置和后置两种写法

  • 前置++i： i先加一，然后参与运算
  • 后置i++: i先参与运算，之后再加一

推荐使用前置版本的写法。

后置递增运算符优先级高于解引用运算符，故*p++相当于*(p++)

成员访问运算符

包括.和->

条件运算符

cond? expr1: expr2;
如果cond为真，则执行expr1，否则执行expr2

位运算符

  • 左移运算符<<在右侧插入0
  • 右移运算符>>，如果运算对象是无符号类型在左侧插入0，如果带符号类型则在左侧插入符号位的副本或则和值为0的二进制位。

const

想让变量的值不能被改变时，可以给变量加上const关键字

  • const对象必须初始化
  • const对象一旦创建其值就不能再改变
  • 默认状态下，const对象仅在文件内有效。如果想在多个文件之间共享cosnt对象，必须在变量的定义前添加extern。

const和引用

可以把引用绑定到const对象上，即对常量的引用

可以将const引用绑定到普通非const对象上

允许初始化const引用时使用结果能转换成引用类型的任意表达式作为初始值

const 和指针

pointer to const 常量指针

指向的内容不能修改，但指针本身是一个变量(*和类型名相邻)

const pointer 指针常量

指针本身是一个常量，必须初始化。（*和const关键字相邻）

const和类

  • 在类成员函数后面加上const关键字，表示该函数内部不能改变成员变量的值,也不能调用其他非const成员函数(静态成员除外)
  • const修饰的类对象只能调用const成员函数

顶层const和底层const

  • 顶层const：表示指针本身是一个常量
  • 底层const：表示指针所指的对象是一个常量

decltype

C++11新标准引入的类型提示符，选择并返回操作数的数据类型
decltype(f()) sum = x
sum的类型就是f()的返回类型。
编译器并不实际调用f，二十使用当带调用发生时f的返回值类型作为sum的类型。

  • 如果decltype使用的参数时引用，则结果是引用类型，变量必须初始化

带符号和无符号类型

除了bool，其他整型可以分成带符号的signed和无符号unsigned两种。

  • signed: 正数，零，负数
  • unsigned: 仅表示大于或等于零的值

int,short,long和long long默认是带符号的，通过在类型名前添加unsigned可以得到无符号类型。

选择类型的一些建议

  1. 使用int进行整数运算。 short常常太小，long常常跟int一样大。如果超过了int的范围，选用long long
  2. 算数表达式中不要使用char和bool
  3. 使用double进行浮点运算。 float通常精度不够，且单精度和双精度的计算代价相差不大

字面值常量literal

如42, ‘a’, “string”这些称为字面值常量。
字面值常量对应的数据类型由它的形式和值决定。

八进制 0开头 024
十六进制 0x或0X开头 0x14

  • 八进制和十六进制的字面值类型是能容纳下当前值的int,unsigned int,long,unsigned long, long long, unsigned long long 中尺寸最小的那个
  • 浮点型字面值默认是double，可以加后缀f来标识float类型

复合类型

引用

格式：type&, 其中type是引用类型指向的类型

  • 引用即对象的别名。
  • 引用必须被初始化。在初始化时，引用会与初始值对象绑定在一起，且无法重新绑定另一个对象。
  • 对引用的操作就是对原对象的操作

指针

格式：type*，其中type为指针指向的类型
指针类型存放着指向类型的地址, 获取变量的地址需要使用取地址符&

解引用符*

空指针nullptr

C++11标准中使用一种特数的字面值nullptr来初始化空指针，它可以被转换成任何其他类型的指针

void*指针

void*可以存放任意对象的地址，不能直接操作void*所指的对象
用途：

  • 与别的指针比较
  • 作为函数的输入或输出
  • 赋给另一个void*指针

指针的引用的区别

  1. 指针本质是一个变量，在逻辑上时独立的，引用是某个变量的别名，在逻辑上依附与某个变量，在定义引用时必须初始化

  2. sizeof(引用)得到的时引用变量的大小，sizeof(指针)得到的时指针变量的大小

  3. 编译时指针在符号表对应的地址值时指针变量的地址值，引用对用的是引用对象的地址值

定义复合变量的两种写法

  1. &, *和变量名挨在一起

  2. &, *和类型名挨在一起

两种都合法，但推荐第一种，因为&,*是修饰变量的。

并发控制

发表于 2019-09-27 | 分类于 数据库 |

基于锁的协议

确保隔离性的方法之一就是对数据项的访问以互斥的方式进行。
实现这种需求的方法是只允许事务访问该事务持有锁的数据

锁

  • 共享锁$S$，如果事务$T_i$获得数据$Q$的共享锁，则$T_i$可读但不可写$Q$。其他事务可以给$Q$加共享锁，不能加排他锁
  • 排他锁$X$，如果事务$T_i$获得数据$Q$的排他锁，则$T_i$可读且可写$Q$。其他事务不能给$Q$加锁

锁的授予

  1. 事务向并发控制管理器申请适当的锁
  2. 如果该数据项已经被加上了不相容的锁，则该事务等待，直到所有其他事务持有的不相容锁被释放
  3. 只有并发管理控制器授予事务所需锁后事务才能继续其操作

两阶段封锁协议

保证可串行性的一个协议，要求每个事务分两个阶段提出加锁和解锁申请。

  1. 增长阶段：事务可以获得锁，但不能释放锁
  2. 缩减阶段：事务可以释放锁，但不能获得新锁

在调度中该事务最后加锁的位置（增长阶段的结束点）称为事务的封锁点，多个事务可以根据封锁点进行排序，这个顺序就是事务的一个可串行化顺序。

两阶段封锁协议不能保证不发生死锁。

级联回滚的问题可以通过严格两阶段封锁协议避免。 处理两阶段封锁协议的要求，它还要求事务持有的所有排他锁必须在事务提交后才能释放。
这样可以防止其他事务在未提交前读这些数据。

事务管理

发表于 2019-09-27 | 分类于 数据库 |

概念

事务是构成单一逻辑工作的一系列操作的集合

ACID

事务具有以下特性：

  • 原子性Atomicity：事务不可分割，要么全部执行，要么全部不执行
  • 一致性Consistency：事务必须保证数据库的一致性
  • 隔离性Isolation：多个事务并发执行时，系统保证每个事务都感觉不到其他事务正在并发执行
  • 持久性Durability：事务成功完成后对数据库的改变会保持下去

抽象事务模型

事务处于以下状态之一：

  • 活动的active
  • 部分提交的partially committed
  • 失败的failed
  • 终止的aborted
  • 提交的committed

回滚：

  1. 杀死：逻辑错误
  2. 重启：硬件错误

可串行化

如果强制事务串行地执行，那么很容易就能保持一致性，而并发很可能违背隔离性。
如果在并发执行中，我们能保证所执行的任何调度的效果都跟没有并发执行的调度效果一样，就可以确保数据库的一致性。
这种调度称为可串行化调度

术语

  • 调度：指令在系统中执行的时间顺序

  • 串行调度：同一事务的各个指令紧挨在一起（执行完一个事务再执行下一个事务),如下图的调度$S$

  • 冲突：如果指令$I$和$J$是在相同的数据上操作，并且其中至少有一个是写操作，则称$I$和$J$是冲突的

  • 冲突等价：如果调度$S$经过一些列非冲突指令交换转换成$S’$，则称$S$和$S’$冲突等价

  • 冲突可串行化：如果调度$S$与一个串行调度冲突等价，则称$S$是冲突可串行化的

如何判断调度$S$是否冲突可串行化

构造优先图$G = (V, E)$, $V$是顶点集，$E$是边集
边集由满足下列三个条件之一的$T_i \rightarrow T_j$组成：

  • $T_i$执行$write(Q)$后，$T_j$执行$read(Q)$
  • $T_i$执行$read(Q)$后，$T_j$执行$write(Q)$
  • $T_i$执行$write(Q)$后，$T_j$执行$write(Q)$

可恢复和无级联调度

可恢复调度

当并发执行中事务发生故障，有些调度不能恢复。

例如下图的调度

如果T6在T7提交后发生故障，需要回滚；
而T7读取了T6写入的数据（我们可以说T7依赖于T6），也需要中止；
但T7已经提交了，无法中止。
这样的调度是不可恢复调度

可恢复调度应该满足：对于每对事务T1和T2，如果T2读取了之前由T1写入的数据，那么T1应该先于T2提交。

无级联调度

即使一个调度是可恢复的，如果多对事务之间存在依赖关系，可能引发级联回滚，导致大量的撤销工作。

无级联调度应该满足:对于每对事务T1和T2，如果T2读取了之前由T1写入的数据，那么T1应该在T2这一读操作之前提交

事务隔离性级别

隔离性级别由高到低由：

  1. 可串行化：保证可串行化调度。
  2. 可重复读：只能读取已提交的数据，并且在一个事务两次读取一个数据时，其他事务不得更新该数据。不要求与其他事务可串行化，
  3. 已提交读：只能读取已提交的数据。不要求可重复读。
  4. 未提交读：允许读取未提交的数据。

我们可以通过多种并发控制机制来实现事务的隔离性级别

树和二叉树

发表于 2019-09-27 |

定义

树是$n$个结点的有限集。在任意一颗非空树中：

  1. 有且仅有一个根节点
  2. 其余节点可分为$m$个互不相交的有限集
  3. 其中每个集合本身也是一颗树，称为跟的子树（递归定义）。

基本术语

度

结点拥有的子树数称为结点的度

叶子结点

度为$0$的结点称为叶子结点

内部结点

除了根结点和叶子结点外的其他结点称为内部结点

孩子/双亲/兄弟/祖先/子孙

  1. 结点的子树的根称为该结点的孩子，该结点被称为孩子的双亲
  2. 同一结点的孩子之间称为兄弟
  3. 从跟到该结点所经分支上的所有结点称为该结点的祖先。

层次

根为第一层，根的孩子为第二层。
若某结点在第$i$层，则其孩子在第$i + 1$层

深度/高度

树中结点的最大层次称为树的深度或高度
