2 两数相加

link

题目

给出两个 非空 的链表用来表示两个非负的整数。其中，它们各自的位数是按照 逆序 的方式存储的，并且它们的每个节点只能存储 一位 数字。
如果，我们将这两个数相加起来，则会返回一个新的链表来表示它们的和。
您可以假设除了数字 0 之外，这两个数都不会以 0 开头。

示例

输入：(2 -> 4 -> 3) + (5 -> 6 -> 4)
输出：7 -> 0 -> 8
原因：342 + 465 = 807

注意

测试用例 说明
$l1=[0, 1]$, $l2=[0, 1, 2]$ 两个链表不一样长
$l1=[]$, $l2=[0, 1]$ 有链表为空
$l1=[1]$, $l2=[9, 9]$ 有额外的进位

19 删除链表倒数第N个节点

link

题目

给定一个链表，删除链表的倒数第 n 个节点，并且返回链表的头结点。

示例

给定一个链表: 1->2->3->4->5, 和 n = 2.
当删除了倒数第二个节点后，链表变为 1->2->3->5.

说明

  • 给定的 n 保证是有效的。
  • 尝试使用一趟扫描实现。

思路

采用双指针p1，p2；

  1. p2先向前移动$n+1$步，使得p1和p2之间间隔为n；
  2. p1和p2同时向前移动，当p2到达链表尾部时，p1到达倒数第n+1个节点，即待删除节点的上一个节点
  3. 删除第n个节点

注意

  • 考虑删除头结点的情况
  • 考虑链表只有一个节点的情况

21 合并两个有序链表

link

题目

将两个有序链表合并为一个新的有序链表并返回。新链表是通过拼接给定的两个链表的所有节点组成的。

示例

输入：1->2->4, 1->3->4
输出：1->1->2->3->4->4

思路

递归实现

  1. 如果l1为空，返回l2， 如果l2为空，返回l1；
  2. 把链表除了头结点的剩余部分看成新链表，当前头结点的下一个节点就是新链表与另一个链表合并后返回的头结点
  3. 返回两个链表中较小的头结点

23合并k个排序链表

link

题目

合并 k 个排序链表，返回合并后的排序链表。请分析和描述算法的复杂度。

示例

输入:
[
1->4->5,
1->3->4,
2->6
]
输出: 1->1->2->3->4->4->5->6

思路

1 逐一比较

每次从k个链表的头结点中选择最小加入新链表

  • 时间复杂度 $O(kN)$, 新链表总共有$N$个节点
  • 空间复杂度 $O(1)$, 不分配新空间，只是改变next指针

优化：比较可以用优先队列实现

分治(递归/自上而下)

将k个链表分成两部分，递归合并每部分

  • 时间复杂度$O(Nlogk)$
  • 空间复杂度$O(1)$

141 环形链表

link

题目

给定一个链表，判断链表中是否有环。
为了表示给定链表中的环，我们使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。 如果 pos 是 -1，则在该链表中没有环。

示例

输入：head = [3,2,0,-4], pos = 1
输出：true
解释：链表中有一个环，其尾部连接到第二个节点。

思路

设置快慢两个指针，慢指针每次移动一步，快指针每次移动两步；
如果有环，快指针会绕多n圈后追上慢指针并重合

代码

补充问题

已知一个有环链表，找出环路的开始位置

设从开始位置到环开始位置的长度为$x$，开始位置到快慢指针重合位置的长度为$y$，环的剩余部分长度为$z$;
则慢指针走过的路程为$x+y$；
快指针走过的路程为$x+y+z+y$(假设快指针多走一圈就追上了慢指针)；
由于快指针的速度是慢指针的两倍，可得等式$2(x+y)=x+y+y+z$；
可以推导出$x=z$

所以，我们可以用一个指针指向开始位置，一个指针指向快慢指针重合位置，两个指针用相同速度向前移动，当两个指针重合时到达环开始位置

160 相交链表

link

题目

编写一个程序，找到两个单链表相交的起始节点。

示例

输入：intersectVal = 8, listA = [4,1,8,4,5], listB = [5,0,1,8,4,5], skipA = 2, skipB = 3
输出：Reference of the node with value = 8
输入解释：相交节点的值为 8 （注意，如果两个列表相交则不能为 0）。从各自的表头开始算起，链表 A 为 [4,1,8,4,5]，链表 B 为 [5,0,1,8,4,5]。在 A 中，相交节点前有 2 个节点；在 B 中，相交节点前有 3 个节点。

思路

  1. 使用两个指针p1，p2分别指向两个链表头部
  2. 两个指针同时往前走，当走到链表尾部时，将指针设为另一个链表的头部
  3. 两个指针相遇时指向的节点就是相交节点

206 反转链表

link

题目

反转一个单链表。

示例

输入: 1->2->3->4->5->NULL
输出: 5->4->3->2->1->NULL

思路

递归法

  1. 假设除了头结点外的子序列已经反转，那么反转后的子序列头节点就是完全反转序列的头结点
  2. 将原头结点添加到反转后的子序列尾部
  3. 返回反转后的子序列头节点

迭代法

依次将节点的next指针反转。
需要用到一个指针pre存储前一个节点，反转前需要一个指针tmp暂时存储后一个节点

234 回文链表

link

题目

请判断一个链表是否为回文链表。

示例

输入: 1->2->2->1
输出: true

思路

  1. 利用一快一慢两个指针，slow指针每次移动一步，fast指针每次移动两部，当fast指针移动到末尾时，slow指针移动到链表中间，将链表就被分成了长度相等两部分
  2. slow指针一边移动一边反转链表
  3. 遍历比较链表两部分是否相等

注意

  • 当链表节点个数为奇数个时，slow最终指向中间节点，且fast->next == null；
  • 当链表节点个数为偶数个时，slow最终指向中间靠后的那个节点，且fast == null；

328 奇偶链表

link

题目

给定一个单链表，把所有的奇数节点和偶数节点分别排在一起。请注意，这里的奇数节点和偶数节点指的是节点编号的奇偶性，而不是节点的值的奇偶性。
请尝试使用原地算法完成。你的算法的空间复杂度应为 O(1)，时间复杂度应为 O(nodes)，nodes 为节点总数。

示例

输入: 1->2->3->4->5->NULL
输出: 1->3->5->2->4->NULL

说明

  • 应当保持奇数节点和偶数节点的相对顺序。
  • 链表的第一个节点视为奇数节点，第二个节点视为偶数节点，以此类推。

思路

使用三个指针：odd指向当前奇数节点，even指向当前偶数节点，evenHead指向偶数序列的起始节点

1.当前奇数节点的下一个节点设为even->next，令奇数节点指针指向下一奇数节点odd = odd->next
2.当前偶数节点的下一节点设为odd->next，令偶数节点指针指向下一偶数节点even = even->next
3.重复上述操作，遍历到链表结尾时结束
4.将奇数序列与偶数序列连接起来odd->next = evenHead

复杂度分析

时间复杂度$O(n)$，空间复杂度$O(1)$
