C++的常用关键字

发表于 2019-10-02 | 分类于 学习笔记 |

const

想让变量的值不能被改变时，可以给变量加上const关键字

  • const对象必须初始化
  • const对象一旦创建其值就不能再改变
  • 默认状态下，const对象仅在文件内有效。如果想在多个文件之间共享cosnt对象，必须在变量的定义前添加extern。

const和引用

可以把引用绑定到const对象上，即对常量的引用

可以将const引用绑定到普通非const对象上

允许初始化const引用时使用结果能转换成引用类型的任意表达式作为初始值

const 和指针

pointer to const 常量指针

指向的内容不能修改，但指针本身是一个变量(*和类型名相邻)

const pointer 指针常量

指针本身是一个常量，必须初始化。（*和const关键字相邻）

const和函数

修饰参数

修饰返回值

const和类

  • 在类成员函数后面加上const关键字，表示该函数内部不能改变成员变量的值,也不能调用其他非const成员函数(静态成员除外)
  • const修饰的类对象只能调用const成员函数

顶层const和底层const

  • 顶层const：表示指针本身是一个常量
  • 底层const：表示指针所指的对象是一个常量

constexpr

常量表达式指的是值不会改变并且在编译过程就能得到计算结果的表达式。

C++11允许将变量声明为constexpr类型，以便编译器验证变量是否是一个常量表达式
声明为constexpr的一定是一个常量。

constexpr函数

constexpr函数指能用与常量表达式的函数。需要满足以下几点：

  • 函数的返回类型以及所有形参类型都是字面值类型
  • 函数体中必须有且只有一条return语句
  • constexpr函数不一定返回常量表达式

const和constexpr的区别

  • const指的是运行时值不变
  • constexpr指的是编译时就能计算出值

static

静态变量的作用范围在文件内，只有本文件内代码可以访问，在其他文件中不可见。

局部变量声明为static

将局部变量声明为static，只有第一次调用函数时执行初始化。

类的静态成员

在类的成员声明之前加上static将其声明为静态成员。
类的静态成员只属于类本身，与类的对象无关。或者说，所有类的对象共享静态成员
静态成员函数不与任何对象绑定在一起，不含有*this指针。

  1. 不能在类内部初始化静态数据成员。
    由于静态数据成员不属于类的任何一个对象，所以他们不是类的构造函数初始化的。

类似于全局变量，静态数据成员定义在所有函数体外，一旦被定义，将一直存在程序的整个声明周期中。

  1. 在外部定义静态成员函数时不能重复static关键字，该关键字只能出现在类内部的声明语句

  2. 静态成员函数不能定义为const成员函数

  3. 静态成员函数不能使用this指针(包括隐式和显式地)

  4. 静态成员的类型可以是类本身，普通成员如果要声明为当前类只能声明成类的指针或引用。

  5. 成员函数可以使用静态成员作为默认实参，普通成员不行

typedef

定义类型的别名

auto

C++11新标准引入auto类型说明符，能让编译器通过初始值推算变量的类型。
具体可以看这篇文章

decltype

C++11新标准引入的类型提示符，选择并返回操作数的数据类型
decltype(f()) sum = x
sum的类型就是f()的返回类型。
编译器并不实际调用f，二十使用当带调用发生时f的返回值类型作为sum的类型。

  • 如果decltype使用的参数时一个变量，则结果是该变量类型，包括顶层const和引用。
  • 如果decltype使用的参数时引用，则结果是引用类型，变量必须初始化

注意，如果加上括号，则编译器会把它当成一个表达式
