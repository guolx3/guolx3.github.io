C++的内联函数

发表于 2019-09-22 | 分类于 学习笔记 |

在函数体前加上inline关键字可以将函数设置为内联函数。
在编译时，编译器会将调用内联函数的地方替换成内联函数体。
内联函数的作用是用空间换时间，牺牲一定的内存来节省函数调用的开销。

注意

  1. 内联函数中不能使用循环和switch
  2. 内联函数不能递归调用自己
  3. 内联函数不能调用其他函数
  4. 代码行数较少且调用频繁的函数推荐使用内联函数
  5. inline关键字只是对编译器的一种建议，被inline修饰的函数不一定被内联。如果函数体过大，编译器会放弃内联而把代码当成普通函数

与宏定义的区别

  • 宏在编译之前完成文本替换，内联函数在编译时被处理
  • 内联函数本质是函数，可以像普通函数一样调用，不像宏定义那样需要加上括号来防止歧义
  • 最好用内联函数来代替宏定义

内存管理4——虚拟内存

发表于 2019-09-17 | 分类于 学习笔记 |

之前讨论的几种管理方式都具有一下两种特征：

  • 一次性。 作业必须一次性全部装入内存后才开始运行，导致当作业过大时不能被全部装入
  • 驻留性。 作业装入内存后，一直驻留在内存中，运行中的进程会因为等待I/O而被阻塞，可能长期处于等待状态

为此，可以采用虚存技术，使得在进程运行时并不需要所有页（段）都在内存中，只在需要的时候读取页（段）。

虚拟内存的实现需要一定的硬件支持，一般包括：

  • 一定容量的内存和外存。
  • 页表机制（段表机制）。
  • 中断机构。当用户程序要访问的部分尚未调入内存时，则产生中断
  • 地址变换机构。逻辑地址到物理地址的转换

局部性原理

  • 时间局部性。 执行的指令可能在不久后再次执行，被访问的数据可能在不久后再次被访问。 原因之一时程序中存在大量的循环语句
  • 空间局部性。 一旦程序访问了某存储单元，则其附近的存储单元也可能被访问。 原因之一时指令通常时顺序存放的，数据也一般是以数组，表等形式存在

请求分页管理方式

请求分页管理方式在基本分页管理方式的基础上增加了请求调页功能和页面置换功能。
这是目前最常用的一种实现虚拟存储器的方法

页表机制

  • 状态位P：指示该页是否已经调入内存
  • 访问字段A：记录访问相关的信息，例如一段时间内被访问的次数，共置换算法参考
  • 修改位M：标识该页在调入内存后是否被修改过
  • 外存地址：用于指出该页在外存的地址，通常时物理块号。

缺页中断机制

每当要访问的页面不在内存中时，就产生一次缺页中断。

内存管理2——连续分配管理方式

发表于 2019-09-15 | 分类于 学习笔记 |

单一连续分配

将内存分为系统区和用户区。系统区供操作系统使用，处于低地址。用户区是除了系统区外的内存空间。
优点

  • 简单
  • 无外部碎片

缺点

  • 只适合单用户的操作系统
  • 有内部碎片
  • 存储器的利用率低

固定分区

最简单的多道程序存储管理方式。
将内存划分成若干个固定大小的区域，区域大小可以相等或者按规则递增，每个区域只能放入一个程序。
为了方便内存分配，需要一张分区说明表，每行记录分区的大小、起始地址和分配状态。分配内存时查找该表寻找合适的分区，如果没找到则不分配内存

优点

  • 简单
  • 处理开销小

缺点

  • 有内部碎片
  • 不能实现多进程共享一个主存区

目前基本没有操作系统使用固定分区的方法

动态分区

不预先划分内存，而是在进程装入内存时根据进程的大小动态建立分区，使得分区大小刚好适合进程大小

缺点

  • 时间长了有外部碎片，可以用紧凑技术对进程进行移动和整理

分配策略

如果同时又多个空闲块满足要求，需要采用一定的分配策略来决定分配哪一块

  1. 首次适应First Fit。按照地址顺序查找到的第一个合适分区。最简单、最好、最快。但是低地址会出现小的空闲分区，而每次重头开始查找都会经过这些分区，增加了查找的开销。
  2. 最佳适应Best Fit。将空闲分区按照容量递增排序，顺序查找到的第一个合适分区。
  3. 最坏适应Worst Fit。将空闲分区按照容量递增排序，顺序查找到的第一个合适分区。
  4. 邻近适应Next Fit。由首次适应演变而来，不同之处是从上一次查找结束的位置开始继续查找。它试图解决首次适应算法低地址空闲分区的问题，但会导致末尾的最大空闲块被分裂成小碎片。

好的程度：首次适应>邻近适应>最佳适应>最坏适应

以上三种分区管理方式的共同点是用户进程在主存中都是连续存储的

内存管理1——基本概念

发表于 2019-09-15 | 分类于 学习笔记 |

为什么要进行内存管理

不可能将所有的进程和所需的资源全都放入内存，所以需要操作系统对内存进行合理的划分和动态管理。

内存管理的功能

  1. 内存空间的分配和回收
  2. 地址转换，将逻辑地址转换为物理地址
  3. 内存的扩展，利用虚拟存储技术或自动覆盖技术从逻辑上扩展内存
  4. 保护，保证各道作业互不干扰

内存管理的基本原理和要求

程序的装入和链接

将源程序转换为可以在内存中执行的进程有以下3步：

  1. 编译。将源程序编译成若干目标模块
  2. 链接。将若干模块以及需要的库函数链接成一个完整的装入模块
  3. 装入。将装入模块装入内存

链接

按照链接的时机不同，可以分成：

  • 静态链接
  • 装入时链接
  • 运行时链接

装入

  • 绝对装入。已经知道程序应该装入的绝对地址，按照地址装入程序和数据，只适用于单道程序设计。该地址可以在编译或汇编时指出或由程序员赋予
  • 可重定位装入。多道程序环境中进程的起始地址通常为0，其他地址是相对地址；在装入时根据相对地址进行地址变换，一次分配完整的内存，如果内存不够则不装入。运行中不能移动或申请内存。
  • 动态装入，将地址转换推迟到运行时再进行。需要重定位寄存器的支持，将内存分配到不连续的存储区，运行时可以动态申请分配内存，可以提供一个比存储空间大的地址空间。

逻辑地址和物理地址

每个目标模块从0开始编址，这些相对地址即逻辑地址；
物理地址是内存中物理单元的真实地址；
将逻辑地址转换为物理地址称为地址重定位

内存保护

  • 保护操作系统不受用户进程的影响
  • 保护用户进程不受其他用户进程的影响

内存保护有两种方法：

  • 采用上下限寄存器记录用户作业在主存中的上下限地址，每次访问地址都进行比较判断是否越界
  • 采用重定位寄存器（基址寄存器）和界地址寄存器（限长寄存器）。重定位寄存器记录最小物理地址，界地址寄存器记录最大物理地址，每次都于界地址寄存器进行比较，如果没有越界则加上重定位寄存器的值映射成物理地址

扩充内存的方法——覆盖和交换

覆盖

程序运行时不可能每时每刻都用到程序和数据的所有部分，所以我们可以把经常访问的部分放在固定区，其它的按照调用关系分段，要用的时候再调入覆盖区

交换

交换有两个基本操作：

  • 换出：将处于等待状态的程序从内存移到辅存
  • 换入：将就绪的程序从辅存移到内存

覆盖主要用于同一个程序，交换用于不同的进程

内存管理3——分页和分段

发表于 2019-09-15 | 分类于 学习笔记 |

非连续分配管理允许将程序装入不相邻的内存空间，需要额外的空间存储索引。
根据分区的大小是否固定，可以分为分页和分段。

基本术语

  • 页框Frame：内存中的一个固定长度的块
  • 页Page： 固定长度的数据块，存储在磁盘，可以临时复制到页框
  • 段：变长的数据块，可以复制到内存中的一个可用区域

分页

把主存空间划分成大小相等的，占用空间比较小的块（页框），进程也划分成较小块（页）；
进程执行时，以块为单位逐个申请主存中的块空间。
不会产生外部碎片，只有在为进程最后一个块分配空间时产生少量的内部碎片（平均只产生半个块大小）

页面的大小

  • 为了方便地址转换，页面的大小为2的幂。
  • 页面过小：导致页表过长，占用大量内存；增加逻辑地址转换为物理地址的开销
  • 页面过大：导致内部碎片增多，降低内存利用率

地址结构

地址结构 = 页号 + 页内偏移量

12~31 11~0
页号P 页内偏移量
  • 地址长度为32位
  • 0 ~ 11位：页内偏移量，说明页大小为$2^{12}$，即4KB
  • 12~31位：页号，说明内存最多允许$2^{20}$页

地址结构决定了虚拟内存的寻址空间有多大

页表

操作系统为每个进程维护一张页表，页表记录页面对应的物理块号，存放在内存中。
页表项 = 页号 + 物理地址

程序未执行时，页表的起始地址和长度存放在进程控制块中，进程执行时存入页表寄存器

地址变换机构

逻辑地址$A$转换为物理地址$E$的步骤：

  1. 计算页号$P = A / L$，偏移量$W = A % L$（$L$为页面长度）
  2. 如果$P \geq M$，则产生越界终端，否则继续执行（$M$为页表长度）
  3. 根据页号取得页表起始地址$F$，计算$对应的页表项地址 = F + P \times 页表项长度$
  4. 根据页表项地址，取出页表中相应的物理块号$b$
  5. 计算物理地址$E = b \times L + W$

存在的两个问题

  1. 每次访存操作都要进行地址转换，地址转换的速度限制了访存速度
  2. 页表太大会使内存利用率降低

快表

如果所有的页表都存放在内存中，那么每一条指令至少要访问两次内存（第一次访问页表获取物理块号和第二次访问计算出来的物理地址）
为了提高指令执行速度，可以使用快表——一个高速缓冲存储器，用来缓存物理块号。
根据计算得出的页号，先在快表中查找物理快号；
如果没有再在页表中查找，并将找到的物理块号存入快表，方便下一次查询。

快表基于局部性原理设计，命中率可达到$90%$

两级页表

为了解决页表过大的问题，可以采用二级页表，即构造一个页表的页表
顶级页表最多只能由一个页面

分段

按照用户进程的自然段划分逻辑空间，长度不固定

逻辑地址

逻辑地址 = 段号S + 段内偏移量W

31~16 15~0
段号S 段内偏移量W

分页中的逻辑地址对用户不可见，分段中的逻辑地址需要用户显式提供（一般由编译程序完成）

段表

每一个进程都由一张段表
段表项 = 段长 + 本段在主存中的起始地址

地址变换机构

从逻辑地址$A$变换为物理地址$E$的步骤如下：

  1. 取出$A$的前几位为段号$S$，后几位为偏移量$W$
  2. 如果$S \geq M$，产生越界终端，否则继续执行（$M$为段表长度）
  3. 取出段表起址$F$，段长$C$, 计算对应的$段表项地址 = F + S \times 段表项长度$。 如果$W \times C$, 发生越界终端。
  4. 根据段表项地址，取出该段的物理起址$b$，计算物理地址$E = b + W$

段的共享与保护

通过两个作业中段表的相应表项指向被共享的段的同一个物理副本来实现

分页是从计算机角度考虑，目的是提高内存利用率，提高计算机性能，完全由硬件实现，对用户透明；
分段考虑了用户，以满足方便编程，信息保护和共享，动态增长和动态链接等多方面的需求

段页式管理方式

分页可以提高内存利用率，分段则能反应程序的逻辑结构并有利于段的共享，两者结合起来形成了段页式管理方式

程序的地址空间先被分成若干逻辑段，每段再分成固定大小的页；
内存空间分成若干和页面大小相同的存储块，以存储块为单位分配内存

逻辑地址

逻辑地址 = 段号S + 页号P + 页内偏移量

每个进程有一张段表（段号 + 页表长度 + 页表起始地址），每个分段有一张页表（页号 + 物理块号）

进行一次访问需要三次访问主存,同样可以用快表来提高查找速度

C++中的类型转换

发表于 2019-09-10 |

在某些情况下，类型转换会自动执行无需程序员借入，这种类型转换称为隐式转换；程序员也可以显式的将对象转换为另一种类型，这种方法称为强制类型转换cast

隐式转换

发生隐式转换的场景有：

  1. 初始化过程中，初始值转换成变量的类型；赋值语句中，右侧运算对象转换称左侧运算对象类型。

    • 当赋给无符号类型的初始值超出范围时，它的实际值是初始值对该类型总大小取模之后的余数。
    • 当赋给有符号类型的初始值超出范围时，它的实际值是未定义的

  2. 整数提升。将小整数类型转换成较大的整数类型。转换后的类型要能容纳原类型所有可能的值

  3. 在条件中，非布尔值转换成布尔类型

  4. 如果算术运算或关系运算有多种类型则转换成同一种类型，尽量不损失精度

  5. 数组转成指针。 在大多数数组的表达式中，数组自动转换成指向数组首元素的指针

注意，当数组被用作decltype的参数，或者作为取地址符&，sizeof及typeid的运算对象时，上述转换不会发生。

  1. 转换成常量。允许指向非常量的指针和引用，允许转换成指向相应的常量类型的指针。

显式转换/强制转换

cast-name<type>(expression)

  • type: 要转换成的目标类型
  • expression： 要转换的值
  • cast-name：包括static_cast，dynamic_cast， const_cast， reinterpret_cast

static_cast

任何具有明确定义的类型转换，只要不包含底层const。

使用场景：当把一个大的算术类型赋值给较小的类型，使用statci_cast告诉读者和编译器这里进行强制转换不在乎精度损失。

dynamic_cast

可以用来转换指针或引用。

使用形式有:

  • dynamic_cast<type*>(e)
  • dynamic_cast<type&>(e)
  • dynamic_cast<type&&>(e)

其中type必须是一个类，且通常含有虚函数；
e的类型应该是目标type或它的基类或它的派生类；
如果是指针类型转换失败则返回0，如果是引用类型转换失败则抛出异常

指针类型的转换

将Base指针bp转换为指向Derived的指针。

我们可以对一个空指针执行dynamic_cast,结果是所需类型的空指针

引用类型的转换

由于不存在空引用，所以当转换失败时，会抛出std::bad_cast的异常

const_cast

可以去掉对象的const性质（也可以加上const性质，但很少用）。

使用场景：

  • 想将一个const的值传进一个非const参数的函数
  • 有函数重载的上下文中

reinterpret_cast

将一个类型转换成无关的其他类型，新值与原始参数有完全相同的比特位。

使用reinterpret_cast非常危险，容易引发糟糕的后果

旧的强制类型转换

  • type(e) //函数风格
  • (type)e //C语言风格

缺点：从表现形式上不清晰明了，不利于调试

运输层的两个主要协议——UDP和TCP

发表于 2019-09-03 | 分类于 学习笔记 |

概述

  • 运输层提供端到端通信，向上面的应用层提供服务。端到端通信指应用进程之间的通信
  • 运输层向高层用户屏蔽了下面的网络核心的细节
  • 两个重要的功能：
    1. 复用：发送方不同的应用进程可以用同一个运输层协议传送数据
    2. 分用：接收方可以从同样的运输层协议数据中分离数据正确交付给目的应用进程

与网络层协议的主要区别

端口

端口号用于识别计算机应用层中的各个进程，TCP/IP用一个16位(最多有65535个)端口号来标记一个端口。
分为两大类：

  1. 服务端使用的端口号
    • 熟知端口号/系统端口号，0~1023。
    • 登记端口号，1024~49151，使用时需要在IANA按手续登记，以防止重复。
  2. 客户端使用端口号，49152~65535，留给客户进程选择暂时使用，通信结束后释放

用户数据报协议UDP

  • 无连接，发送数据之前不需要建立连接

10 正则表达式匹配

link

题目

给你一个字符串 $s$ 和一个字符规律 $p$，请你来实现一个支持 . 和 * 的正则表达式匹配。
. 匹配任意单个字符

  • 匹配零个或多个前面的那一个元素
    s 可能为空，且只包含从 a-z 的小写字母。
    p 可能为空，且只包含从 a-z 的小写字母，以及字符 . 和 * 。

示例

输入:
s = “aa”
p = “a”
输出: false
解释: “a” 无法匹配 “aa” 整个字符串。

思路

记字符串$s$的长度为$ls$, 字符串$p$的长度为$lp$; 记动态规划网络为dp, 为了用下标$0$表示字符为空的情况，状态表的大小应该为$(ls + 1) \times (lp + 1)$

状态定义

dp[i][j]表示s[0, i-1]与p[0, j - 1]是否匹配。其中s[0, i-1]表示字符串$s$从下标$0$到下标$i - 1$的子串，p[0, j - 1]同理

状态初始化：

  1. 如果$s$, $p$都为空则判定为匹配，即dp[0][0] = true。
  2. 如果$s$为空，且p[j - 1] == '*'，那么dp[0][j] = dp[0][j - 2](即下标$j$处的*匹配零个字符)

状态转换

  1. 如果p[j - 1] == '*'
    • 如果s[i - 1] == p[j - 2] || p[j - 2] == '.', 那么dp[i][j] = dp[i - 1][j]（匹配一个或多个）
    • 否则dp[i][j] = dp[i][j - 2](匹配零个)
  2. 否则
    如果p[j - 1] == '.' || s[i - 1] == p[j - 1]), 那么dp[i][j] = dp[i - 1][j - 1]

44 通配符匹配

link

题目

给定一个字符串 (s) 和一个字符模式 (p) ，实现一个支持 ‘?’ 和 ‘*’ 的通配符匹配。

‘?’ 可以匹配任何单个字符。
‘*’ 可以匹配任意字符串（包括空字符串）。

示例

输入:
s = “aa”
p = “*”
输出: true
解释: ‘*’ 可以匹配任意字符串。

思路

状态定义

定义bool match[i][j]表示字符串s[0, i - 1]与p[0, j - 1]是否匹配。
（match用下标$0$表示空字符串的情况）

状态转换

  • 当p[j] != '*', match[i][j] = match[i - 1][j - 1] && (s[i - 1] == p[j - 1] || p[j - 1] == '？')
  • 否则，可能存在下面三种情况
    • match[i][j] = match[i][j - 1]（星号匹配空字符串)
    • match[i][j] = match[i - 1][j] (星号匹配一个字符)

状态初始化

  1. match[0][0] = true
  2. match[0][j] = match[0][j - 1] && p[j - 1] == '*', 即如果$s$为空，$p$不为空，只有$p$全是星号才匹配
